# GEANT4 Easy Install Scripts

Bash scripts for downloading, building, and installing Geant4 and related
libraries on Linux. The repository intentionally keeps the install logic in
git, while third-party source trees, build directories, archives, and install
outputs are downloaded or generated by the scripts and ignored by git.

## Repository Layout

- `installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash` is the main
  unified Ubuntu installer for Geant4 10.7.4 with Qt/OpenGL GUI support,
  GDML, G3TOG4, and multithreading enabled.
- `installation_script_Ubuntu_20_04_FullGeant4_10_7_multithreaded.bash`,
  `installation_script_Ubuntu_22_04_FullGeant4_10_7_multithreaded.bash`, and
  `installation_script_Ubuntu_24_04_FullGeant4_10_7_multithreaded.bash` are
  compatibility wrappers around the unified installer. Each wrapper checks
  that it is running on the matching Ubuntu release.
- `installation_script_linuxOnly_CADmesh.bash` installs CADMesh against the
  Geant4 10.7.4 installation produced by this repository.
- `installation_script_linuxNoGUI_Geant4_10_7_multithreaded.bash` keeps the
  older no-GUI Linux workflow.
- `installation_script_linuxOnly_matio_hdf5_zlib.bash` installs zlib, HDF5, and
  matio.
- `docker_easy_deployment/` contains the Docker-based deployment workflow.
- `OLD/` contains older installers kept for reference.

## Ubuntu Geant4 10.7.4 Install

Run the unified installer from the repository root:

```bash
./installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash
```

The script currently supports Ubuntu 20.04, 22.04, and 24.04. It detects the
Ubuntu version from `/etc/os-release`, installs missing apt dependencies after
interactive confirmation, downloads CMake, Xerces-C, and Geant4 sources, then
builds into the repo-local ignored `geant4/` directory.

You can also use a version-specific wrapper:

```bash
./installation_script_Ubuntu_22_04_FullGeant4_10_7_multithreaded.bash
```

Useful options:

```bash
./installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash --dry-run
./installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash --jobs 4
./installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash --clean
./installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash --skip-deps
./installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash --no-update-shell-rc
./installation_script_Ubuntu_FullGeant4_10_7_multithreaded.bash --work-dir /path/to/workdir
```

After a successful install, the script writes:

```text
geant4/geant4_10_7_env.sh
```

By default it also adds a managed block to `~/.bashrc`. Open a new terminal or
run:

```bash
source ~/.bashrc
```

## CADMesh Install

Install Geant4 first, then run:

```bash
./installation_script_linuxOnly_CADmesh.bash
```

Useful options:

```bash
./installation_script_linuxOnly_CADmesh.bash --dry-run
./installation_script_linuxOnly_CADmesh.bash --jobs 4
./installation_script_linuxOnly_CADmesh.bash --no-update-shell-rc
./installation_script_linuxOnly_CADmesh.bash --base-dir /path/to/geant4-workdir
```

The CADMesh installer expects the Geant4 environment file and install directory
created by the Ubuntu Geant4 10.7.4 installer. It writes:

```text
geant4/cadmesh_env.sh
```

By default it also adds a managed CADMesh block to `~/.bashrc`.

## Docker Deployment

Docker deployment is kept in git under `docker_easy_deployment/`:

```bash
cd docker_easy_deployment
docker compose build
docker compose up
```

The Docker image is based on Ubuntu 22.04 and builds Geant4 10.7.4, Xerces-C
3.2.2, and CADMesh during the image build. See
`docker_easy_deployment/README.md` for the Docker-specific workflow.

## Generated Files

The scripts download and generate large files locally. These are ignored by git,
including:

- `geant4/`
- downloaded source archives such as `*.tar.gz`
- downloaded source directories such as `xerces-c-*` and `source_geant4.*`
- build directories such as `geant4_build_*` and `build_xercesc_g4_*`
- install directories such as `geant4_install_*` and `install_xercesc_g4_*`

This keeps the repository focused on Bash scripts, documentation, and Docker
deployment files.

## Notes

- Run these scripts with `bash` or execute them directly; do not run them with
  `sh`.
- The full Ubuntu installer may need `sudo` to install missing apt packages.
- The bundled CMake download currently expects an x86_64 Linux machine.
- Geant4 and CADMesh builds can take a long time. Use `--jobs N` to control
  parallelism on machines with limited memory.
